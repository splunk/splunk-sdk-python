data/inputs/ad

Creates new or modifies existing performance monitoring settings.

[POST] data/inputs/ad

Parameters

disabled
Required string Indicates whether the monitoring is disabled.
index
Required string The index in which to store the gathered data.
monitorSubtree
Required string Whether or not to monitor the subtree(s) of a given directory tree path. 1 means yes, 0 means no.
name
Required string A unique name that represents a configuration or set of configurations for a specific domain controller (DC).
startingNode
Required string Where in the Active Directory directory tree to start monitoring. If not specified, will attempt to start at the root of the directory tree.
targetDc
Required string Specifies a fully qualified domain name of a valid, network-accessible DC. If not specified, Splunk will obtain the local computer’s DC.

Status Codes

Status Code Description
201 Created successfully.
500 Internal server error. See response body for details.
403 Insufficient permissions to create monitoring stanza.
402 The Splunk license in use has disabled this feature.
401 Authentication failure: must pass valid credentials with request.
400 Request error. See response body for details.
503 This feature has been disabled in Splunk configuration files.
409 Request error: this operation is invalid for this item. See response body for details.

data/inputs/monitor

Create a new file or directory monitor input.

[POST] data/inputs/monitor

Parameters

blacklist
Required string Specify a regular expression for a file path. The file path that matches this regular expression is not indexed.
check-index
Required string If set to true, the “index” value will be checked to ensure that it is the name of a valid index.
check-path
Required string If set to true, the “name” value will be checked to ensure that it exists.
crc-salt
Required string A string that modifies the file tracking identity for files in this input. The magic value “" invokes special behavior (see admin documentation).
followTail
Required string If set to true, files that are seen for the first time will be read from the end.
host
Required string The value to populate in the host field for events from this data input.
host_regex
Required string Specify a regular expression for a file path. If the path for a file matches this regular expression, the captured value is used to populate the host field for events from this data input. The regular expression must have one capture group.
host_segment
Required string Use the specified slash-separate segment of the filepath as the host field value.
ignore-older-than
Required string Specify a time value. If the modification time of a file being monitored falls outside of this rolling time window, the file is no longer being monitored.
index
Required string Which index events from this input should be stored in.
name
Required string The file or directory path to monitor on the system.
recursive
Required string Setting this to “false” will prevent monitoring of any subdirectories encountered within this data input.
rename-source
Required string The value to populate in the source field for events from this data input. The same source should not be used for multiple data inputs.
sourcetype
Required string The value to populate in the sourcetype field for incoming events.
time-before-close
Required string When Splunk reaches the end of a file that is being read, the file will be kept open for a minimum of the number of seconds specified in this value. After this period has elapsed, the file will be checked again for more data.
whitelist
Required string Specify a regular expression for a file path. Only file paths that match this regular expression are indexed.

Status Codes

Status Code Description
201 Created successfully.
500 Internal server error. See response body for details.
403 Insufficient permissions to create monitored input.
402 The Splunk license in use has disabled this feature.
401 Authentication failure: must pass valid credentials with request.
400 Request error. See response body for details.
503 This feature has been disabled in Splunk configuration files.
409 Request error: this operation is invalid for this item. See response body for details.

Creates of modifies existing event log collection settings. You can configure both native and WMI collection with this endpoint.

[POST] data/inputs/win-event-log-collections

Parameters

hosts
Required string A comma-separated list of addtional hosts to be used for monitoring. The first host should be specified with “lookup_host”, and the additional ones using this parameter.
index
Required string The index in which to store the gathered data.
logs
Required string A comma-separated list of event log names to gather data from.
lookup_host
Required string This is a host from which we will monitor log events. To specify additional hosts to be monitored via WMI, use the “hosts” parameter.
name
Required string This is the name of the collection. This name will appear in configuration file, as well as the source and the sourcetype of the indexed data. If the value is “localhost”, it will use native event log collection; otherwise, it will use WMI.

Status Codes

Status Code Description
201 Created successfully.
403 Insufficient permissions to create event log collections.
402 The Splunk license in use has disabled this feature.
401 Authentication failure: must pass valid credentials with request.
400 Request error. See response body for details.
409 Request error: this operation is invalid for this item. See response body for details.
503 This feature has been disabled in Splunk configuration files.
500 Internal server error. See response body for details.

receivers/stream

Create events from the stream of data following HTTP headers.

[POST] receivers/stream

Parameters

<data_stream>
Required string Raw event text. This does not need to be presented as a complete HTTP request, but can be streamed in as data is available.
host
Required string The value to populate in the host field for events from this data input.
host_regex
Required string A regular expression used to extract the host value from each event.
index
Required string The index to send events from this input to.
source
Required string The source value to fill in the metadata for this input’s events.
sourcetype
Required string The sourcetype to apply to events from this input.

Status Codes

Status Code Description
200 Data accepted.
404 Receiver does not exist.
400 Request error. See response body for details.

Example

Streams an arbitrary number of events to Splunk. This example is best demonstrated via a program rather than a curl request. Below is a short python script that runs indefinitely, until the user presses Ctrl-C. In the meantime, it will send one event per second to Splunk. Note that for a streaming connection, the “x-splunk-input-mode” header must be specified.

Request

import httplib, time
conn = httplib.HTTPSConnection("localhost", 8089)
conn.connect()
conn.putrequest("POST", "/services/receivers/stream?source=www&sourcetype=web_data")
conn.putheader("Authorization", "Basic YWRtaW46cGFzcw&#61;&#61;")
conn.putheader("x-splunk-input-mode", "streaming")
conn.endheaders()
print "Looping..."
while True:
    conn.send("%s A sample event.\n" % time.asctime())
    time.sleep(1)

Response
